# captcha-copilot 需求文档

> AI Copilot for CAPTCHA — 当自动化被验证码卡住时，自动识别并处理，让工作流继续

## 一、项目概述

### 1.1 项目名称
captcha-copilot

### 1.2 项目定位
开源、免费、纯 AI 视觉驱动的验证码自动处理 skill。作为 AI agent 的即插即用技能，当自动化流程被验证码阻断时，自动识别验证码类型并执行对应操作，恢复工作流。

### 1.3 核心原则
- **零成本** — 纯 AI 视觉识别，不依赖第三方付费打码平台
- **开箱即用** — 用户添加到 AI agent 技能栏即可使用，无需额外配置
- **隐私优先** — 只截取验证码区域，不截取全页敏感信息
- **经验积累** — 记住每个网站的反爬特征，越用越聪明

### 1.4 目标用户
- AI agent 自动化用户（browser-use、Playwright、Puppeteer 等）
- 数据抓取/信息获取场景的 AI 辅助用户
- 任何需要 AI agent 处理网页自动化任务的开发者

## 二、功能需求

### 2.1 第一层：预防 — 反爬规避

#### 2.1.1 反爬触发条件检测

检测网站的三类反爬触发条件：

| 类型 | 检测项 | 说明 |
|------|--------|------|
| 频率类 | 请求频率 | 单位时间内请求/点击次数过多 |
| 频率类 | 操作间隔 | 操作间隔过于均匀，不像人类 |
| 行为类 | 页面停留 | 停留时间过短即执行操作 |
| 行为类 | 鼠标痕迹 | 缺少鼠标移动、滚动等人类痕迹 |
| 行为类 | 直接操作 | 直接操作元素无前置移动轨迹 |
| 指纹类 | WebDriver | 浏览器 WebDriver 特征被检测 |
| 指纹类 | User-Agent | 请求头异常 |
| 指纹类 | 环境指纹 | 浏览器环境指纹不一致 |

#### 2.1.2 经验记忆机制

- 记录每个网站对什么行为敏感（记行为类型，不记硬数字）
- 记忆结构示例：
  ```
  {
    "example.com": {
      "sensitivity": ["frequency", "mouse_trajectory"],
      "strategy": ["slow_down", "simulate_mouse_move"],
      "last_updated": "2026-04-15"
    }
  }
  ```
- 阈值策略不固定，网站策略调整后经验可覆盖更新
- 支持社区贡献经验数据

#### 2.1.3 主动规避策略

| 策略 | 适用场景 | 具体行为 |
|------|----------|----------|
| 降速 | 频率敏感站点 | 操作间隔随机化（2-8秒） |
| 模拟鼠标移动 | 行为检测站点 | 操作前增加自然的鼠标移动轨迹 |
| 模拟滚动 | 行为检测站点 | 操作前模拟页面滚动 |
| 随机停留 | 行为检测站点 | 页面加载后随机等待 1-3 秒 |
| 指纹处理 | 指纹检测站点 | 隐藏 WebDriver 特征、修改 User-Agent |
| 人类轨迹模拟 | 滑块/拖拽场景 | 加速-减速-微调的拖拽曲线 |

---

### 2.2 第二层：检测 — 验证码识别

#### 2.2.1 规则匹配检测（优先）

通过 DOM 特征匹配已知验证码组件：

| 验证码类型 | DOM 特征 |
|------------|----------|
| reCAPTCHA v2 | iframe[src*="recaptcha"], .g-recaptcha, div[data-sitekey] |
| reCAPTCHA v3 | 同 v2，额外检测 grecaptcha.execute() |
| hCaptcha | iframe[src*="hcaptcha"], .h-captcha, div[data-hcaptcha-sitekey] |
| Cloudflare Turnstile | .cf-turnstile, div[data-turnstile-sitekey] |
| 滑块验证码 | 常见 class/id: slider, drag, verify, captcha-slider |
| 点选验证码 | 常见 class/id: click-captcha, select-text, verify-click |
| 旋转验证码 | 常见 class/id: rotate-captcha, captcha-rotate |
| 字符验证码 | img[src*="captcha"], canvas + input 组合 |

#### 2.2.2 AI 视觉兜底检测

规则未命中时，使用 AI 视觉模型判断页面是否存在验证码：

- 截取当前页面截图
- 发送给 AI 视觉模型，提问："当前页面是否存在验证码？如果存在，是什么类型？"
- 支持的识别类型：
  1. 字符扭曲识别
  2. 数学计算型
  3. 滑块拼图
  4. 点选文字/图标
  5. 旋转验证
  6. reCAPTCHA v2（图片九宫格）
  7. reCAPTCHA v3（行为评分）
  8. hCaptcha
  9. Cloudflare Turnstile
  10. 空间推理型
  11. 拼图还原型

#### 2.2.3 验证码区域定位

- 规则匹配时：通过 DOM 定位验证码元素坐标
- 视觉检测时：由 AI 视觉模型标注验证码区域边界
- 隐私保护：只截取验证码区域，不截取全页

---

### 2.3 第三层：破解 — 验证码处理

#### 2.3.1 各类型处理方案

| 验证码类型 | 识别方式 | 操作方式 | 预期成功率 |
|------------|----------|----------|------------|
| 字符扭曲 | AI 视觉 OCR | 输入识别结果到文本框 | 85-90% |
| 数学计算 | AI 视觉识别算式 | 计算结果后输入 | 90-95% |
| 滑块拼图 | AI 视觉定位缺口 | 模拟人类拖拽轨迹 | 70-80% |
| 点选文字 | AI 视觉定位坐标 | 按顺序点击 | 60-75% |
| 点选图标 | AI 视觉识别+定位 | 点击目标图标 | 60-75% |
| 旋转验证 | AI 视觉判断角度 | 模拟旋转拖拽 | 65-75% |
| reCAPTCHA v2 | AI 视觉识别九宫格 | 点击复选框+图片选择 | 60-70% |
| reCAPTCHA v3 | 行为评分优化 | 调整操作模式 | 不适用（预防层处理） |
| hCaptcha | AI 视觉识别九宫格 | 点击目标图片 | 55-70% |
| Turnstile | 环境检测规避 | 指纹处理+等待 | 40-60% |
| 空间推理 | AI 视觉推理 | 点击目标元素 | 60-70% |
| 拼图还原 | AI 视觉匹配 | 拖拽碎片到正确位置 | 55-65% |

#### 2.3.2 滑块拖拽轨迹模拟

人类拖拽特征：
- 起步加速（前 20% 距离加速）
- 中段匀速（中间 50% 匀速偏移）
- 末段减速（后 20% 减速）
- 微调回弹（到达目标后小幅回弹再校正）
- Y 轴偏移（人类拖拽不是完美水平线）
- 耗时随机（800-2000ms）

#### 2.3.3 多轮验证码支持

- 每轮处理完成后，检测是否还有下一轮
- 最多支持 5 轮，超出则放弃并通知用户
- 每轮之间随机等待 1-3 秒

#### 2.3.4 通过确认机制

验证码提交后，判断是否通过：

| 信号 | 含义 |
|------|------|
| 验证码弹窗消失 | 通过 |
| 页面刷新/跳转 | 通过 |
| 出现"验证成功"类提示 | 通过 |
| 出现"验证失败/请重试"类提示 | 未通过 |
| 弹窗仍在且内容变化 | 多轮验证，继续 |
| 无变化（5秒超时） | 状态未知，视为未通过 |

#### 2.3.5 智能重试策略

- 最大重试次数：3 次
- 重试间隔：递增（2s → 4s → 6s）
- 重试前刷新验证码（重新获取）
- 连续 3 次失败则放弃，通知用户并询问是否手动处理
- 记录失败信息到经验库

---

### 2.4 通用机制

#### 2.4.1 任务中断与恢复

- 验证码出现时，保存当前任务状态（执行到哪一步、操作目标是什么）
- 验证码通过后，自动恢复中断点继续执行
- 任务状态结构：
  ```
  {
    "task_id": "uuid",
    "task_description": "用户原始任务描述",
    "interrupted_step": 3,
    "total_steps": 5,
    "context": { ... },
    "timestamp": "2026-04-15T15:30:00Z"
  }
  ```

#### 2.4.2 隐私保护

- 截图时只截取验证码区域（基于 DOM 坐标或 AI 视觉定位的区域边界）
- 不截取全页面
- 截图中如包含用户敏感信息（账号、余额等），由 AI 视觉检测并遮挡
- 截图数据不持久化存储，处理完即销毁

#### 2.4.3 人类行为模拟

- 鼠标移动：贝塞尔曲线轨迹，非直线
- 点击前：先移动到目标附近，短暂停留，再点击
- 操作间隔：随机 500-3000ms
- 页面滚动：随机滚动幅度和速度
- 输入文字：逐字符输入，间隔 50-200ms

## 三、版本规划

### v0.1 — MVP（最小可用版本）

**目标：覆盖 60% 常见场景，跑通核心流程**

功能范围：
- 规则匹配检测（覆盖 reCAPTCHA、hCaptcha、常见滑块/点选）
- AI 视觉兜底检测
- 三种验证码处理：字符扭曲识别、滑块拼图、点选文字
- 基础人类行为模拟（鼠标轨迹、操作间隔随机化）
- 通过确认机制
- 基础重试策略（最多 3 次）
- 任务中断与恢复

不包含：
- 预防层（反爬规避）
- 经验记忆
- 旋转验证、reCAPTCHA 九宫格、Turnstile 等复杂类型

### v0.2 — 预防层

**目标：从"被卡住再处理"升级到"主动避免被卡住"**

新增功能：
- 反爬触发条件检测（频率、行为、指纹三类）
- 经验记忆机制（网站敏感行为记录 + 规避策略自动应用）
- 主动规避策略（降速、模拟鼠标、指纹处理）
- 指纹处理（隐藏 WebDriver、修改 User-Agent）

### v0.3 — 复杂验证码支持

**目标：覆盖 90%+ 验证码类型**

新增功能：
- reCAPTCHA v2 九宫格处理
- hCaptcha 九宫格处理
- 旋转验证处理
- 空间推理型处理
- 拼图还原型处理
- 多轮验证码支持
- 增强的滑块轨迹模拟

### v0.4 — 社区化

**目标：借助社区力量持续进化**

新增功能：
- 经验库社区贡献机制（提交网站反爬特征）
- 规则库社区贡献机制（提交新验证码 DOM 特征）
- 验证码样本数据集开放
- 成功率统计与反馈系统
- Turnstile 等难点的专项优化

## 五、非功能需求

### 5.1 安全性
- 截图数据内存中处理，不落盘
- 经验库数据本地存储，用户可控
- 不记录用户网站凭证

### 5.2 可扩展性
- 验证码类型处理器可插拔（新增类型只需实现统一接口）
- 规则库支持热更新
- 经验库支持导入/导出

### 5.3 可维护性
- 每种验证码类型独立模块
- 清晰的日志输出（检测、识别、操作、结果）
- 配置项可调（重试次数、超时时间、操作速度等）

## 六、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| AI 视觉识别准确率不稳定 | 用户体验差，放弃使用 | 分阶段发布，明确标注各类型成功率；社区持续优化 |
| 网站反爬策略更新 | 规则库失效 | 社区贡献机制，快速响应更新 |
| GitHub TOS 限制 | 项目被下架 | 定位为"AI 辅助自动化"，避免破解/绕过等敏感措辞 |
| Turnstile 等纯环境检测型 | 这类验证码无法通过视觉解决 | 预防层重点处理，标注已知限制 |
| 滑块轨迹被检测 | 验证码操作正确但仍被拒 | 持续优化轨迹算法，增加随机性和自然度 |
